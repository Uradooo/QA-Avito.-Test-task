# QA Avito - Тестовое задание

Task 1 - задание на тестирование веб-страницы (поиск багов, анализ приоритетов и оформление баг-репортов).
Task 2 - задание на тестирование API (разработка тест-кейсов, автоматизация тестов и оформление баг-репортов).

## Структура проекта

```
Task1_API/
├── Bugs.png                     # изображение найденных дефектов
└── Task1.md                     # описание дефектов

Task2_API/
├── tests/
│   ├── conftest.py              # общие фикстуры для pytest
│   ├── test_create_item.py      # тесты создания объявлений
│   ├── test_get_item.py         # тесты получения объявления по ID
│   ├── test_get_statistic.py    # тесты получения статистики объявления
│   └── test_get_seller_items.py # тесты получения списка объявлений продавца
├── utils/
│   ├── api_client.py            # HTTP-клиент для отправки запросов
│   ├── assertions.py            # вспомогательные проверки (assert'ы)
│   └── data_generator.py        # генератор тестовых данных (рандомные названия, цены и т.д.)
├── config.py                    # настройки (URL, таймауты, заголовки)
├── pytest.ini                   # конфигурация pytest (флаги, маркеры)
├── requirements.txt             # зависимости проекта (requests, pytest и др.)
├── TESTCASES.md                 # описание тест-кейсов
└── BUGS.md                      # найденные баги с описанием и шагами воспроизведения
```

## Установка и запуск

### 1. Установить зависимости

```bash
cd Task2_API
pip install -r requirements.txt
```

### 2. Запустить все тесты

```bash
pytest

```

## Дополнительно

- `sellerId` генерируется в диапазоне 111111-999999 (как требует задание).
- Часть негативных тестов ожидаемо падает, они описаны в `BUGS.md`.
- Тесты независимы: каждый создаёт и удаляет свои данные через фикстуру `created_item`.
